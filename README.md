# Fin Research Agent

**面向专业投研团队的金融研究 Agent** — 多源异构数据融合 + 矛盾识别与交叉验证 + 可溯源结构化输出

## 为什么要做？

**真问题，不是伪需求**：专业投研团队 60%+ 时间花在"找信息"而非"做判断"。

| 现状痛点 | 真实后果 |
|----------|----------|
| 信息散落 5+ 平台 | 反复切换系统，错过最佳决策窗口 |
| 人工交叉验证 | 关键矛盾被遗漏，事后才发现"漏看了那条专家观点" |
| 结论无法溯源 | 投决会被问"依据是什么"，说不清楚 |

**不是换壳，是新范式**：
- 传统方案 = AI 搜索引擎（给一个答案）
- 我们的方案 = AI 初级研究员（多方收集 → 分类整理 → 标注矛盾 → 追溯来源 → 辅助判断）

## 为什么是现在？

| 时机因素 | 具体表现 |
|----------|----------|
| **技术刚好够用** | LLM 推理能力可胜任"论点提取 + 矛盾识别"，RAG 解决金融文档检索难题 |
| **市场窗口期** | 买方降本增效压力大，合规要求投资决策有据可查 |
| **数据基础就绪** | 专家访谈/研报/社交媒体等数据源已 API 化，知识库产品降低接入门槛 |

## 解决的核心痛点

专业投研团队做投资研究时面临的困境：

| 痛点 | 具体表现 |
|------|----------|
| **信息极度分散** | 业绩会纪要在一个平台、专家观点在另一个平台、社交媒体实时动态又在别处，每次做投资判断都要在多个渠道间反复切换 |
| **效率低且易遗漏** | 人工交叉比对耗时耗力，关键信息容易遗漏，错过最佳决策窗口 |
| **时效性风险** | 上季度的结论可能已被最新业绩推翻，过时信息导致错误决策 |
| **矛盾性难处理** | 不同来源对同一公司的判断可能完全相反，缺乏系统化手段识别和处理这些矛盾 |

## 我们的方案

构建一个面向专业投研团队的金融数据分析 Agent，对接业绩会纪要、专家访谈、研报、社交媒体等多源数据，通过**意图识别自动路由**到最相关的信息源，用**两步检索策略**（宽召回 + 精准回答）解决金融文档标题与内容弱关联的检索难题，并对多源信息做**交叉验证、矛盾标注和时效性判断**，最终以**来源可追溯**的方式输出结论，形成可沉淀的研究底稿。

## 为什么是 AI Agent？

这个场景天然需要 Agent 能力——**不是单轮问答，而是需要自主规划检索路径、跨多个数据源执行操作、对结果做推理整合**。

| 传统搜索工具 | AI Agent |
|-------------|----------|
| 只能逐个平台查询 | 一次对话完成全流程 |
| 需要人工判断去哪查 | 自动识别意图，路由到最佳数据源 |
| 结果需要人工比对 | 自动交叉验证，标注矛盾 |
| 无法处理信息冲突 | 结构化输出多空观点，辅助决策 |

**Agent 完整决策链路**: 判断意图 → 多源检索 → 交叉验证 → 矛盾标注 → 结构化输出

## 核心创新点

### 1. 多源异构数据融合
对接 **5 类金融信息源**，覆盖投研全场景：

| 数据源 | 内容类型 | 独特价值 |
|--------|---------|----------|
| 专家访谈 | 行业专家一对一访谈 | 一手信息、产业洞察、非公开观点 |
| AlphaEngine | 研报、业绩说明会纪要 | 官方数据、分析师评级、财务指标 |
| Twitter | KOL 动态、市场情绪 | 实时舆情、事件驱动、情绪信号 |
| Substack | 独立分析师 Newsletter | 深度研究、另类视角、长期观点 |
| AceCampTech | 社区共享调研纪要 | 广覆盖、及时性、多元观点 |

### 2. 两步检索策略
解决金融文档「标题与内容弱关联」的检索难题：

- **Stage 1 - 宽召回**: 问题原文 + 公司名变体扩展，最大化召回相关文档
- **Stage 2 - 精准回答**: 基于召回结果提取关键实体，精准定位答案

### 3. 意图识别 + 智能路由
根据问题类型自动选择最优数据源组合：

| 问题类型 | 路由策略 |
|----------|----------|
| 业绩会查询 | AlphaEngine 优先 |
| 专家观点 | 专家访谈优先 |
| 行业趋势 | 专家访谈 + AceCampTech |
| 实时舆情 | Twitter + Substack |

### 4. 矛盾识别与交叉验证
对多源信息做系统化处理：
- **时效性判断**: 自动识别信息新旧，标注可能过时的结论
- **矛盾标注**: 当不同来源观点冲突时，明确标注分歧点
- **交叉验证**: 多源信息互相印证，提升结论可信度

### 5. Bull/Bear 结构化输出
不给模糊答案，提供**可对比的多空观点**，每个观点可追溯到原始来源：

```
🐂 看多论点                         🐻 看空论点
├─ AI服务器强劲增长                  ├─ 传统PC业务持续承压
│  强度: 85% | 来源: 专家访谈         │  强度: 78% | 来源: AceCampTech
├─ 与NVIDIA深度合作                  ├─ AI服务器毛利率偏低
│  强度: 72% | 来源: AlphaEngine     │  强度: 65% | 来源: 专家访谈
└─ 企业AI市场领先地位                └─ 竞争加剧(HPE/联想)
   强度: 68% | 来源: Twitter            强度: 70% | 来源: 研报

⚖️ 综合判断: HOLD | 置信度: 68% | 多空比: 55:45
⚠️ 矛盾标注: 专家访谈与研报对AI服务器毛利率判断存在分歧
```

## 目标用户与商业价值

### 核心用户：买方投研团队（资管/基金公司）

| 维度 | 内容 |
|------|------|
| **谁** | 基金公司/资管公司的研究员、分析师、基金经理助理 |
| **什么情况下** | 投资决策前的信息收集、持仓公司定期跟踪、行业专题研究 |
| **干什么** | 快速整合业绩会、专家访谈、研报、舆情等多源信息，识别市场分歧点，形成可溯源的研究底稿 |
| **核心价值** | 研究效率提升 3-5 倍，避免遗漏关键信息，降低因信息时效性/矛盾性导致的决策失误 |

**典型场景**: 某基金持仓 Dell，财报发布后，研究员需要在 1 小时内整合业绩会纪要、卖方研报、专家访谈、Twitter 舆情，形成多空观点对比，供基金经理决策。

### 次要用户：卖方研究部门（券商/投行）

| 维度 | 内容 |
|------|------|
| **谁** | 券商研究所的行业研究员 |
| **什么情况下** | 撰写深度研报前、覆盖新公司时、行业重大事件发生时 |
| **干什么** | 快速了解市场已有观点、识别观点分歧、避免重复研究、找到差异化切入点 |
| **核心价值** | 缩短研报准备周期，提供差异化观点依据 |

**典型场景**: 研究员准备覆盖 NVIDIA，需要快速了解市场对其 AI 业务的多空分歧在哪，找到自己研报的差异化切入点。

### 潜在用户：金融数据服务商

| 维度 | 内容 |
|------|------|
| **谁** | Wind/Bloomberg/同花顺等金融信息终端的产品团队 |
| **什么情况下** | 需要为现有产品增加智能投研能力 |
| **干什么** | 将 Agent 能力作为增值模块嵌入现有数据终端 |
| **核心价值** | 提升产品竞争力，增加客户粘性 |

## 快速开始

### 1. 克隆仓库
```bash
git clone https://github.com/YOUR_USERNAME/fin-research-agent.git
cd fin-research-agent
```

### 2. 安装依赖
```bash
pip install -r requirements.txt
```

### 3. 配置环境变量（可选）
```bash
cp .env.example .env
# 编辑 .env 文件，填入 API Key
```

> **注意**: 不配置环境变量时，系统自动使用 Demo 模式（内置 mock 数据）

### 4. 运行
```bash
streamlit run demo/app.py
```

访问 http://localhost:8501
